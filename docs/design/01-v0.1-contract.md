# v0.1 Contract

The first real package target is a simple univariate Gaussian animal model.

## Planned Julia Capability

Phase 1 should support:

- pedigree validation and sorting: implemented as an engine utility;
- ID recoding: implemented as an engine utility;
- founder and unknown-parent handling: implemented as an engine utility;
- direct sparse `Ainv`: implemented as an engine utility;
- in-memory phenotype/pedigree/genotype/expression data container:
  implemented as `HSData`, with conservative ID-map diagnostics;
- low-level animal-model spec validation: implemented for `y`, `X`, `Z`,
  `Ainv`, IDs, Gaussian family, and ML/REML method;
- Gaussian ML/REML likelihood evaluation: implemented as a dense experimental
  path at supplied variance components, with a `max_dense_cells` guard;
- variance-component optimization: implemented as a dense experimental path for
  validated Julia specs;
- EBVs/BLUPs: implemented as dense experimental extractors for validated Julia
  specs;
- variance components: implemented as a dense experimental extractor;
- heritability: implemented as a dense experimental univariate extractor;
- prediction error variance and reliability: implemented as dense experimental
  extractors for validated Julia specs;
- fixed-effect design from public formulas: parsed in R, not bridged into Julia
  execution yet;
- random-effect design from public formulas: parsed in R, not bridged into
  Julia execution yet;
- sparse production REML/ML fitting and AI-REML;
- production sparse reliability and prediction error variance.

## R Surface

The R twin now parses the narrow v0.1 grammar:

```r
hsquared(
  y ~ sex + age + animal(1 | id, pedigree = ped),
  data = dat,
  family = gaussian()
)
```

As of R head `b57b48e`, `hsquared` validates this formula and builds an
internal `hs_bridge_payload`, then stops honestly at the Julia bridge boundary.
As of R head `9eabf0d`, users can opt into the experimental tiny/local Julia
path with:

```r
hsquared(..., control = hs_control(engine = "julia"))
```

The default remains `hs_control(engine = "validate")`, which parses, validates,
builds the payload, and stops.

As of R head `644c75e`, `hsquared` also has an `hs_data()` container for
phenotype, pedigree, genotype, expression, marker, annotation, and environment
inputs. Julia mirrors the in-memory ID-map vocabulary with `HSData`, but live
R-to-Julia data-container marshalling is still not implemented.

The R payload contains `y`, dense `X`, sparse `Z`, method, family, normalized
IDs, normalized pedigree rows with parent indices, and metadata. `Ainv` is
deliberately `NULL` on the R side and marked `ainv_status = "build_in_julia"`.

The intended Julia target is:

```julia
fit_animal_model(spec::AnimalModelSpec)
```

or the lower-level payload-equivalent call:

```julia
fit_animal_model(y, X, Z, Ainv; method = :REML)
```

Both Julia calls are implemented as experimental dense paths with a
`max_dense_cells` safety guard. The missing piece is R-side sparse `Z`
marshalling using Julia's `sparse_csc_matrix()` helper, stable production engine
controls, and Mrode validation.

## Syntax-Parity Rule

The R `hsquared` surface is the authority for applied users. The Julia engine
should accept the same model intent through a bridge payload, and any future
direct Julia formula-like syntax should stay as close to the R spelling as the
language allows.

Allowed discrepancies:

- Julia type names such as `Gaussian()` where R uses `gaussian()`;
- Julia boolean spelling such as `true` where R uses `TRUE`;
- explicit Julia engine objects when users bypass the R bridge.

Disallowed drift:

- Julia fitting a model that R cannot describe in the shared contract;
- R claiming a Julia engine capability that is not implemented and validated;
- different meanings for the same syntax token.

## Engine Boundary

Julia owns sparse relationship construction and fitting. R owns formula
capture, data validation, user-facing messages, S3 methods, and extractors.

## Gate

v0.1 requires a Mrode simple animal-model example, the now-present
hand-checked tiny pedigree tests, and at least one comparator check where
available.
