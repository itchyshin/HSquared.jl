## Goal

Turn the construction-only genomic utilities (`genomic_relationship_matrix`,
`genomic_relationship_inverse`) into an actual fitted GBLUP solve, reusing the
existing Phase-1 Henderson MME — engine-internal, additive, and validatable
comparator-free.

## Key Finding (drove the design)

A VanRaden `G` is **always** rank-deficient: column-centering puts the all-ones
vector in `G`'s null space (`1ᵀW = 0`), so `rank(G) ≤ n − 1` regardless of marker
count. GBLUP therefore always needs a regularized `Ginv` (the existing
`genomic_relationship_inverse` ridge). This also means the GBLUP↔SNP-BLUP
equivalence must route the GBLUP side through the marginal `V = σ²g·G + σ²e·I`
(never invert `G`) — deferred to the SNP-BLUP slice.

## Implementation

`fit_gblup(y, X, Z, Ginv, sigma_a2, sigma_e2; ids, method)` is a thin wrapper:
`animal_model_spec(y, X, Z, Ginv; ...)` then `henderson_mme(spec, ...)`. The
genomic precision enters the same `Ainv` slot the pedigree animal model uses
(`AnimalModelSpec.Ainv::TQ<:AbstractMatrix`; `_sparse_mme_system` materializes
any `AbstractMatrix`), so GBLUP reuses the validated supplied-variance MME solve
verbatim — no new solver, no new result type, every extractor works by dispatch.

## Public Claim Audit

Allowed: GBLUP supplied-variance solve (experimental) — feeds a regularized
genomic `Ginv` into the validated Henderson MME; reproduces an independent dense
MME (~1e-15) and reproduces pedigree BLUP exactly when `G = A` (~1e-30).

Blocked: REML estimation of genomic variance components, single-step,
marker-effect outputs, external-comparator parity (AGHmatrix/sommer/BLUPF90),
and any production/large-data/sparse-`G` performance claim — the dense `Ginv`
path is validation-scale only and does not gain the selinv sparsity advantage.

## Tests Of The Tests

Invariant 1 (independent dense MME assembled in the test, not via `src`) catches
a wrong MME or extractor; invariant 2 (`G = A` → pedigree BLUP) ties correctness
to the already-covered pedigree path (`V1-MME`); invariant 3 (variance-ratio
invariance) catches a λ scaling error; the pinned β/GEBV are a hand-reproducible
anchor.

## Known Limitations

- Supplied variance components only (no genomic REML yet).
- Dense `Ginv` (`O(q²)`/`O(q³)`); no sparse/APY `G`, no GPU.
- No external comparator (AGHmatrix/sommer/BLUPF90) — R-lane validation debt
  (`V2-GBLUP` / `V5-GBLUP`).
