## What I derived (and where the spec was incomplete)

- Conditional `ℓ = log Φ(sη)`, `s = 2y−1`; score `s·M(sη)` (signed inverse-Mills,
  `M = φ/Φ`); weight `−ℓ″ = M(sη)·(M(sη)+sη)`. I verified the probit is LOG-CONCAVE
  (`M(z)(M(z)+z) ∈ (0,1)`, the conditional variance of a unit normal), so the
  observed information is strictly positive and equals the expected information — no
  Fisher-scoring substitution is needed (contrast beta-binomial H2). Verified
  weight(−6) ≈ 0.94 < 1.
- **Numerics:** dependency-free `Φ` via the package's already-validated incomplete
  gamma (`erfc(z) = Q(½, z²)`), plus a LOG-form continued fraction so `_norm_logcdf`
  stays finite into the deep tail (a naive `log(Φ)` underflows around x ≈ −39). The
  coarse genomic `_standard_normal_cdf_approx` (7.5e-8) is deliberately NOT reused.
- **Spec gap I found:** the spec described a probit VA path as "low-risk via GH
  reuse." Deriving it showed the VA expected information is RESPONSE-DEPENDENT
  (`−E[ℓ″]` varies with `s = 2y−1`), but the existing `_fam_expected_weight(f, ηbar,
  v)` signature carries no `y`. A correct VA path needs a signature generalisation
  across all families. So — consistent with H1/H2 — I shipped **Laplace-only** and
  reject `:variational`, documenting this as explicit follow-up (honest under-promise,
  not an overclaim).
